# トラブルシューティング: 銘柄数が増えない

## 現在の状況
- 表示されている銘柄数: 49銘柄（モックデータ）
- 期待される銘柄数: 1,000銘柄（J-Quants API使用時）

## 確認手順

### 1. ブラウザの開発者ツールでログを確認

1. **開発者ツールを開く**
   - F12キーを押す
   - 「Console」タブを開く

2. **手動更新を実行**
   - スクリーニング画面で「手動更新」ボタンをクリック
   - コンソールに以下のようなログが表示されるか確認：
     ```
     === API Route: POST /api/update ===
     Manual update requested
     API Key: xxxxx...
     API Base: https://api.jquants.com/v1
     Clearing cache with key: ...
     Cache cleared
     Calling fetchStocksFromJQuants...
     Attempting to fetch stocks from J-Quants API...
     ```

3. **エラーメッセージを確認**
   - コンソールにエラーが表示されていないか確認
   - エラーメッセージをコピー

### 2. サーバー側のログを確認

開発サーバーを起動しているターミナル（コマンドプロンプト）で、以下のようなログが表示されるか確認：

```
=== API Route: POST /api/update ===
Manual update requested
API Key: xxxxx...
Clearing cache...
Calling fetchStocksFromJQuants...
Attempting to fetch stocks from J-Quants API...
```

### 3. 設定画面でAPIキーを確認

1. `/settings`にアクセス
2. APIキーが正しく保存されているか確認
3. 「保存」ボタンを再度クリック

### 4. ネットワークタブで確認

1. 開発者ツールの「Network」タブを開く
2. 「手動更新」ボタンをクリック
3. `/api/update` のリクエストを確認
4. リクエストヘッダーに `x-api-key` が含まれているか確認
5. レスポンスを確認

## よくある問題

### 問題1: APIキーが送信されていない

**症状**: コンソールに「API Key: not provided」と表示される

**原因**: 
- localStorageにAPIキーが保存されていない
- リクエストヘッダーにAPIキーが含まれていない

**解決方法**:
1. 設定画面（`/settings`）でAPIキーを再保存
2. ブラウザを再読み込み（F5）
3. 再度「手動更新」を実行

### 問題2: J-Quants APIの認証エラー

**症状**: コンソールに「Failed to get refresh token」や「Failed to get ID token」と表示される

**原因**: 
- J-Quants APIの認証方式が正しくない
- APIキーの形式が間違っている
- メールアドレスとパスワードも必要

**解決方法**:
1. J-Quants APIのドキュメントで認証方式を確認
2. APIキーの形式が正しいか確認
3. 必要に応じて、メールアドレスとパスワードも設定

### 問題3: キャッシュが残っている

**症状**: 古いデータ（49銘柄）が表示される

**解決方法**:
1. 「手動更新」ボタンをクリック（キャッシュをクリア）
2. ブラウザのキャッシュをクリア（Ctrl + Shift + Delete）
3. ハードリロード（Ctrl + F5）

### 問題4: API呼び出し制限に達している

**症状**: 「Rate limit exceeded」エラー

**解決方法**:
- 1時間待ってから再試行
- 無料プランの制限（1時間あたり100回）を確認

### 問題5: J-Quants APIが0件を返す

**症状**: コンソールに「J-Quants API returned 0 stocks」と表示される

**原因**:
- APIキーは有効だが、データが取得できない
- 日付指定が間違っている
- APIの仕様が変更された

**解決方法**:
1. J-Quants APIのドキュメントで最新の仕様を確認
2. APIキーが有効か確認
3. 日付パラメータを確認

## デバッグ情報の収集

問題を特定するために、以下の情報を収集してください：

1. **ブラウザのコンソールログ**
   - エラーメッセージ全体
   - 警告メッセージ

2. **サーバー側のログ**
   - ターミナルに表示されるログ
   - エラーメッセージ

3. **ネットワークタブの情報**
   - `/api/update` のリクエストヘッダー
   - レスポンスの内容

4. **設定情報**
   - APIキーが設定されているか（最初の10文字のみ）
   - APIベースURL

## 次のステップ

上記の確認を行っても解決しない場合：

1. コンソールのログをコピー
2. エラーメッセージを共有
3. J-Quants APIのドキュメントを確認
4. 必要に応じて実装を修正
