# パスワード確認ガイド

## メールアドレスが正しい場合の確認ポイント

メールアドレスが正しい場合でも、以下の点を確認してください：

### 1. パスワードの確認

**通常のJ-Quantsアカウントの場合:**
- J-Quantsアカウント作成時に設定したパスワードを使用
- パスワードが正しいか確認

**Googleアカウント認証の場合:**
- **2段階認証が無効**: Googleアカウントのパスワードを使用
- **2段階認証が有効**: **アプリパスワード**を生成して使用する必要があります

### 2. アプリパスワードの生成方法（Googleアカウント認証の場合）

1. **Googleアカウントのセキュリティ設定を開く**
   - https://myaccount.google.com/security にアクセス
   - または Googleアカウント → セキュリティ

2. **2段階認証を確認**
   - 「2段階認証プロセス」が有効になっているか確認

3. **アプリパスワードを生成**
   - 「2段階認証プロセス」の下に「アプリパスワード」がある
   - 「アプリパスワード」をクリック
   - アプリ名（例: J-Quants API）を入力
   - 「生成」をクリック
   - 表示された16文字のパスワードをコピー

4. **設定画面で使用**
   - メールアドレス: Googleアカウントのメールアドレス
   - パスワード: 生成したアプリパスワード（16文字）

### 3. サーバーログで確認

「手動更新」を実行した後、サーバーログで以下を確認：

1. **認証情報が送信されているか**
   ```
   Email: xxxxx...
   Password: provided
   ```

2. **リフレッシュトークンの取得に成功しているか**
   ```
   ✓ Successfully obtained refresh token
   ```

3. **IDトークンの取得に成功しているか**
   ```
   ✓ Successfully obtained ID token
   ```

4. **エラーが発生していないか**
   - `Failed to get refresh token: 401` → パスワードが間違っている可能性
   - `Failed to get ID token: 401` → リフレッシュトークンの問題
   - `Failed to fetch listed info: 401` → IDトークンの問題

### 4. よくある問題

#### 問題1: 2段階認証が有効なのに通常のパスワードを使用している

**症状**: `Failed to get refresh token: 401` エラー

**解決方法**:
- アプリパスワードを生成して使用

#### 問題2: パスワードに特殊文字が含まれている

**症状**: 認証に失敗する

**解決方法**:
- パスワードをコピー&ペーストで入力
- スペースや改行が含まれていないか確認

#### 問題3: J-Quantsアカウントが有効でない

**症状**: `Failed to get refresh token: 401` エラー

**解決方法**:
- J-Quants公式サイトでログインできるか確認
- アカウントが有効か確認

## 次のステップ

1. **サーバーログを確認**
   - 「手動更新」を実行した時のログ全体を確認
   - 特にエラーメッセージに注意

2. **パスワードを再確認**
   - Googleアカウント認証の場合: アプリパスワードを使用しているか確認
   - 通常のアカウントの場合: パスワードが正しいか確認

3. **設定画面で再入力**
   - メールアドレスとパスワードを再入力
   - 「保存」ボタンをクリック
   - 再度「手動更新」を実行

4. **サーバーログを共有**
   - エラーメッセージがあれば共有してください
   - より具体的な解決策を提案できます
