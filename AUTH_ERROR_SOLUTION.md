# 認証エラー解決ガイド

## 現在のエラー

```
Response status: 400
Error response: {"message": "'mailaddress' or 'password' is incorrect."}
```

## 原因

J-Quants APIが「メールアドレスまたはパスワードが間違っている」と返しています。

メールアドレスが正しい場合、**パスワードが間違っている可能性が高い**です。

## 解決方法

### 方法1: パスワードを確認

1. **J-Quants公式サイトでログインできるか確認**
   - https://jpx-jquants.com/ にアクセス
   - メールアドレスとパスワードでログインできるか確認
   - ログインできない場合: パスワードが間違っているか、アカウントが有効でない

2. **パスワードを再入力**
   - 設定画面（`/settings`）でパスワードを再入力
   - スペースや改行が含まれていないか確認
   - コピー&ペーストで入力することを推奨

### 方法2: Googleアカウント認証の場合

GoogleアカウントでJ-Quantsに登録した場合：

1. **2段階認証が有効か確認**
   - Googleアカウントのセキュリティ設定を確認
   - 2段階認証が有効な場合、**アプリパスワード**が必要です

2. **アプリパスワードを生成**
   - https://myaccount.google.com/security にアクセス
   - 「2段階認証プロセス」の下に「アプリパスワード」がある
   - 「アプリパスワード」をクリック
   - アプリ名（例: J-Quants API）を入力
   - 「生成」をクリック
   - 表示された16文字のパスワードをコピー

3. **設定画面で使用**
   - メールアドレス: `y_sugiura@nextjp.co.jp`
   - パスワード: 生成したアプリパスワード（16文字）

### 方法3: J-Quantsアカウントでメールアドレスを確認

Googleアカウント認証の場合、J-Quantsアカウントにメールアドレスが登録されていない可能性があります。

1. **J-Quants公式サイトにログイン**
   - Googleアカウントでログイン

2. **アカウント設定を確認**
   - プロフィール設定やアカウント設定を開く
   - メールアドレスが登録されているか確認
   - 登録されていない場合、メールアドレスを追加

## 確認手順

1. **設定画面でパスワードを再入力**
   - `/settings` にアクセス
   - パスワードを再入力（コピー&ペースト推奨）
   - 「保存」ボタンをクリック

2. **手動更新を実行**
   - スクリーニング画面で「手動更新」ボタンをクリック

3. **サーバーログを確認**
   - エラーが解消されているか確認
   - `✓ Successfully obtained refresh token` と表示されれば成功

## モックデータについて

現在、認証エラーが発生しているため、モックデータ（49銘柄）にフォールバックしています。

認証が成功すれば、J-Quants APIから1,000銘柄を取得できます。

## 次のステップ

1. **パスワードを確認**
   - J-Quants公式サイトでログインできるか確認
   - Googleアカウント認証の場合: アプリパスワードを生成

2. **設定画面で再入力**
   - パスワードを再入力して保存

3. **手動更新を実行**
   - エラーが解消されているか確認

4. **まだエラーが出る場合**
   - J-Quants公式サイトのお問い合わせフォームから問い合わせ
   - Googleアカウント認証でのAPI使用方法を確認
