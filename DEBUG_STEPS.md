# デバッグ手順: 銘柄数が増えない問題

## 現在の状況
- エラーは表示されていない
- 銘柄数は49銘柄のまま（モックデータ）
- J-Quants APIからデータが取得できていない

## 確認手順

### ステップ1: 設定画面で認証情報を確認

1. **設定画面にアクセス**
   - `/settings` にアクセス
   - または上部ナビゲーションの「設定」をクリック

2. **認証方式を確認**
   - 「認証方式」が選択されているか確認
   - メールアドレス/パスワード方式の場合: メールアドレスとパスワードが入力されているか
   - APIキー方式の場合: APIキーが入力されているか

3. **保存されているか確認**
   - 「保存」ボタンをクリックして保存
   - ブラウザを再読み込み（F5）して、設定が保持されているか確認

### ステップ2: ブラウザのコンソールでログを確認

1. **開発者ツールを開く**
   - F12キーを押す
   - 「Console」タブを開く

2. **手動更新を実行**
   - スクリーニング画面で「手動更新」ボタンをクリック

3. **ログを確認**
   - 以下のようなログが表示されるか確認：
     ```
     === API Route: POST /api/update ===
     Manual update requested
     Email: xxxxx... (または API Key: xxxxx...)
     Calling fetchStocksFromJQuants...
     Attempting to fetch stocks from J-Quants API...
     ```

4. **エラーメッセージを確認**
   - エラーが表示されていないか確認
   - 特に以下のエラーに注意：
     - `Failed to get refresh token`
     - `Failed to get ID token`
     - `J-Quants email/password or API key is required`

### ステップ3: サーバー側のログを確認

開発サーバーを起動しているターミナル（コマンドプロンプト）で、以下のログを確認：

1. **認証情報が送信されているか**
   ```
   Email provided: Yes (xxxxx...)
   または
   API Key provided: Yes (xxxxx...)
   ```

2. **API呼び出しが実行されているか**
   ```
   Attempting to fetch stocks from J-Quants API...
   ```

3. **エラーが発生していないか**
   - エラーメッセージが表示されていないか確認

### ステップ4: ネットワークタブで確認

1. **開発者ツールの「Network」タブを開く**
   - F12 → 「Network」タブ

2. **手動更新を実行**
   - 「手動更新」ボタンをクリック

3. **リクエストを確認**
   - `/api/update` のリクエストをクリック
   - 「Headers」タブでリクエストヘッダーを確認
   - `x-jquants-email` または `x-jquants-api-key` が含まれているか確認

4. **レスポンスを確認**
   - 「Response」タブでレスポンスを確認
   - エラーメッセージが含まれていないか確認

## よくある問題と解決方法

### 問題1: 認証情報が送信されていない

**症状**: ログに「Email provided: No」「API Key provided: No」と表示される

**原因**: 
- localStorageに認証情報が保存されていない
- 認証方式が正しく選択されていない

**解決方法**:
1. 設定画面で認証情報を再入力
2. 「保存」ボタンをクリック
3. ブラウザを再読み込み（F5）
4. 再度「手動更新」を実行

### 問題2: 認証は成功しているが、データが取得できない

**症状**: ログに「✓ Successfully obtained refresh token」「✓ Successfully obtained ID token」と表示されるが、銘柄数が0

**原因**:
- J-Quants APIのエンドポイントが正しくない
- APIのレスポンス形式が想定と異なる
- データ取得のロジックに問題がある

**解決方法**:
1. J-Quants APIのドキュメントでエンドポイントを確認
2. 実際のAPIレスポンスを確認（Networkタブで確認）
3. 実装を修正

### 問題3: エラーが発生しているが、モックデータにフォールバックしている

**症状**: エラーログが表示されるが、アプリは動作している（49銘柄が表示される）

**原因**:
- J-Quants APIの認証に失敗
- エラーハンドリングでモックデータにフォールバックしている

**解決方法**:
1. エラーメッセージを確認
2. 認証情報が正しいか確認
3. J-Quants APIの仕様を確認

## 次のステップ

上記の確認を行っても解決しない場合：

1. **コンソールのログ全体をコピー**
   - エラーメッセージを含むすべてのログ

2. **サーバー側のログをコピー**
   - ターミナルに表示されるすべてのログ

3. **Networkタブの情報を確認**
   - `/api/update` のリクエストとレスポンス

4. **設定情報を確認**
   - 使用している認証方式（メールアドレス/パスワード or APIキー）
   - 認証情報の最初の数文字（セキュリティのため）

これらの情報を共有していただければ、より具体的な解決策を提案できます。
