# クイックチェック: 銘柄数が増えない問題

## 即座に確認すべき3つのポイント

### 1. 設定画面で認証情報が入力されているか

**確認方法:**
1. `/settings` にアクセス
2. 以下を確認：
   - 「認証方式」が「メールアドレス/パスワード（推奨）」になっている
   - 「メールアドレス」欄にメールアドレスが入力されている
   - 「パスワード」欄にパスワードが入力されている
   - 「保存」ボタンをクリックして保存

**重要**: 設定画面で「保存」ボタンをクリックしないと、認証情報は保存されません。

### 2. サーバーログで認証情報が送信されているか

**確認方法:**
1. 開発サーバーを起動しているターミナル（コマンドプロンプト）を確認
2. スクリーニング画面で「手動更新」ボタンをクリック
3. ターミナルに以下のログが表示されるか確認：
   ```
   === API Route: POST /api/update ===
   Email: xxxxx... (メールアドレスの最初の5文字)
   Password: provided
   ```

**問題がある場合:**
- `Email: not provided` と表示される → 設定画面で認証情報を再入力して保存
- `Password: not provided` と表示される → 設定画面で認証情報を再入力して保存

### 3. 認証エラーが発生していないか

**確認方法:**
サーバーログで以下のエラーが出ていないか確認：

1. **リフレッシュトークンの取得に失敗**
   ```
   Failed to get refresh token: 401
   ```
   → メールアドレスまたはパスワードが間違っている
   → 正しい認証情報を入力しているか確認

2. **IDトークンの取得に失敗**
   ```
   Failed to get ID token: 401
   ```
   → リフレッシュトークンの有効期限が切れている
   → 再度「手動更新」を実行

3. **データ取得に失敗**
   ```
   Failed to fetch listed info: 401
   ```
   → IDトークンが無効または期限切れ
   → 再度「手動更新」を実行

## 最も可能性が高い原因

### 原因1: 設定画面で認証情報を保存していない

**症状**: サーバーログに「Email: not provided」と表示される

**解決方法**:
1. 設定画面（`/settings`）にアクセス
2. メールアドレスとパスワードを入力
3. **必ず「保存」ボタンをクリック**
4. ブラウザを再読み込み（F5）
5. 再度「手動更新」を実行

### 原因2: メールアドレス/パスワードが間違っている

**症状**: サーバーログに「Failed to get refresh token: 401」と表示される

**解決方法**:
1. J-Quantsアカウントのメールアドレスとパスワードを確認
2. Googleアカウント認証の場合:
   - 2段階認証が有効な場合は、アプリパスワードを生成
   - アプリパスワードの生成方法: Googleアカウントのセキュリティ設定 → アプリパスワード
3. 設定画面で正しい認証情報を入力して保存
4. 再度「手動更新」を実行

### 原因3: キャッシュが残っている

**症状**: 認証情報を更新したが、古いデータが表示される

**解決方法**:
1. 「手動更新」ボタンをクリック（キャッシュをクリア）
2. ブラウザのキャッシュをクリア（Ctrl + Shift + Delete）
3. ハードリロード（Ctrl + F5）

## 次のステップ

上記の確認を行った後、以下を共有してください：

1. **サーバーログ**（「手動更新」を実行した時のログ全体）
2. **設定画面の状態**（メールアドレス/パスワードが入力されているか）
3. **エラーメッセージ**（あれば）

これらの情報があれば、より具体的な解決策を提案できます。
