# 認証情報の確認手順

## 現在の状況
- 銘柄数: 49銘柄（モックデータ）
- 目標: 1,000銘柄（J-Quants API使用時）

## 確認手順

### ステップ1: 設定画面で認証情報を確認

1. **設定画面にアクセス**
   - 上部ナビゲーションの「設定」をクリック
   - または http://localhost:3000/settings に直接アクセス

2. **認証方式を確認**
   - 「認証方式」が「メールアドレス/パスワード（推奨）」になっているか確認

3. **メールアドレスとパスワードを入力**
   - **メールアドレス**: J-Quantsアカウントのメールアドレス
     - Googleアカウント認証の場合: Googleアカウントのメールアドレス（例: `yourname@gmail.com`）
   - **パスワード**: J-Quantsアカウントのパスワード
     - Googleアカウント認証の場合:
       - 2段階認証が無効: Googleアカウントのパスワード
       - 2段階認証が有効: アプリパスワード（Googleアカウントのセキュリティ設定で生成）

4. **保存**
   - 「保存」ボタンをクリック
   - 「✓ 認証情報を保存しました」と表示されれば成功

### ステップ2: 手動更新を実行

1. **スクリーニング画面に戻る**
   - 上部ナビゲーションの「スクリーニング」をクリック

2. **手動更新を実行**
   - 「手動更新」ボタンをクリック

3. **サーバーログを確認**
   - 開発サーバーを起動しているターミナル（コマンドプロンプト）で、以下のログを確認：
     ```
     === API Route: POST /api/update ===
     Manual update requested
     Email: xxxxx... (メールアドレスの最初の5文字)
     Password: provided
     ```

### ステップ3: エラーログを確認

サーバーログで以下のエラーが出ていないか確認：

1. **認証情報が送信されていない**
   ```
   Email: not provided
   Password: not provided
   ```
   → 設定画面で認証情報を再入力して保存

2. **リフレッシュトークンの取得に失敗**
   ```
   Failed to get refresh token: 401
   ```
   → メールアドレスまたはパスワードが間違っている可能性
   → 正しい認証情報を入力しているか確認

3. **IDトークンの取得に失敗**
   ```
   Failed to get ID token: 401
   ```
   → リフレッシュトークンの有効期限が切れている可能性
   → 再度「手動更新」を実行

4. **データ取得に失敗**
   ```
   Failed to fetch listed info: 401
   ```
   → IDトークンが無効または期限切れ
   → 再度「手動更新」を実行

## よくある問題

### 問題1: 認証情報が保存されていない

**症状**: サーバーログに「Email: not provided」と表示される

**解決方法**:
1. 設定画面で認証情報を再入力
2. 「保存」ボタンをクリック
3. ブラウザを再読み込み（F5）
4. 再度「手動更新」を実行

### 問題2: メールアドレス/パスワードが間違っている

**症状**: サーバーログに「Failed to get refresh token: 401」と表示される

**解決方法**:
1. J-Quantsアカウントのメールアドレスとパスワードを確認
2. Googleアカウント認証の場合:
   - 2段階認証が有効な場合は、アプリパスワードを生成して使用
   - アプリパスワードの生成方法: Googleアカウントのセキュリティ設定 → アプリパスワード

### 問題3: 認証は成功しているが、データが取得できない

**症状**: サーバーログに「✓ Successfully obtained refresh token」「✓ Successfully obtained ID token」と表示されるが、銘柄数が0

**解決方法**:
1. J-Quants APIの制限を確認（無料プランでは1,000銘柄まで）
2. API呼び出し制限を確認（1時間あたり100回）
3. サーバーログでエラーメッセージを確認

## 次のステップ

上記の確認を行っても解決しない場合：

1. **サーバーログ全体をコピー**
   - 「手動更新」を実行した時のログ全体

2. **設定情報を確認**
   - 使用している認証方式
   - メールアドレスの最初の数文字（セキュリティのため）

3. **エラーメッセージを共有**
   - サーバーログに表示されるエラーメッセージ

これらの情報を共有していただければ、より具体的な解決策を提案できます。
