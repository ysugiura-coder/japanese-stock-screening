# APIキー設定後のデバッグ手順

## 問題: 銘柄数が増えない

### 確認手順

1. **ブラウザの開発者ツールを開く**
   - F12キーを押す
   - 「Console」タブを開く

2. **設定画面でAPIキーを確認**
   - `/settings`にアクセス
   - APIキーが正しく保存されているか確認

3. **手動更新を実行**
   - スクリーニング画面に戻る
   - 「手動更新」ボタンをクリック
   - コンソールに以下のようなログが表示されるか確認：
     ```
     Attempting to fetch stocks from J-Quants API...
     API Key: xxxxx...
     API Base: https://api.jquants.com/v1
     ```

4. **エラーログを確認**
   - コンソールにエラーメッセージが表示されていないか確認
   - よくあるエラー：
     - `Failed to get refresh token`
     - `Failed to get ID token`
     - `Failed to fetch listed info`

5. **サーバー側のログを確認**
   - ターミナル（コマンドプロンプト）で開発サーバーを起動している場合
   - サーバー側のログも確認

## よくある問題と解決方法

### 問題1: APIキーの形式が正しくない

**症状**: `Failed to get refresh token` エラー

**原因**: J-Quants APIの認証方式が2つあります：
1. メールアドレス/パスワード方式
2. APIキー方式（直接使用）

**解決方法**:
- J-Quants APIのドキュメントで認証方式を確認
- 設定画面でメールアドレスとパスワードも設定する必要がある場合があります

### 問題2: キャッシュが残っている

**症状**: 古いデータが表示される

**解決方法**:
1. ブラウザのキャッシュをクリア
2. 「手動更新」ボタンをクリック（キャッシュをクリアして再取得）

### 問題3: APIキーが送信されていない

**症状**: モックデータ（49銘柄）が表示される

**確認方法**:
1. 開発者ツールの「Network」タブを開く
2. `/api/stocks` または `/api/update` のリクエストを確認
3. リクエストヘッダーに `x-api-key` が含まれているか確認

**解決方法**:
- 設定画面でAPIキーを再保存
- ブラウザを再読み込み

### 問題4: J-Quants APIの制限に達している

**症状**: `Rate limit exceeded` エラー

**解決方法**:
- 1時間待ってから再試行
- 無料プランの制限（1時間あたり100回）を確認

## デバッグ用のログ追加

以下のログが追加されました：

1. **APIキーの確認**
   - APIキーが設定されているか
   - APIキーの最初の10文字（セキュリティのため）

2. **認証フローのログ**
   - リフレッシュトークン取得
   - IDトークン取得
   - 各ステップでのエラー

3. **データ取得のログ**
   - 取得した銘柄数
   - エラーメッセージの詳細

## 次のステップ

1. コンソールのログを確認
2. エラーメッセージをコピー
3. J-Quants APIのドキュメントで認証方式を確認
4. 必要に応じて実装を修正
